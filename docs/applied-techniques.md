# Applied Techniques

## 1. Large Language Models (LLMs)

Large Language Models are deep neural networks trained on massive text corpora that exhibit strong natural language understanding and generation capabilities. In this project, OpenAI's GPT-4o-mini serves as the core inference engine, responsible for interpreting user queries in Vietnamese legal language and generating coherent, contextually appropriate responses. The system leverages the model's chat completion API with configurable temperature and token limits, and supports both synchronous and streaming inference modes; the latter delivers tokens incrementally to the client via Socket.io `stream_chunk` events, reducing perceived latency. Furthermore, the LLM operates within a multi-turn conversation framework where up to six prior exchanges are maintained as context, enabling the model to resolve anaphoric references and follow-up questions across a session.

## 2. Retrieval-Augmented Generation (RAG)

Retrieval-Augmented Generation is a paradigm that mitigates the hallucination problem inherent in LLMs by grounding their output in externally retrieved evidence. In this system, when a legal question is received, the user query is first embedded into a dense vector using OpenAI's `text-embedding-3-small` model and then used to perform approximate nearest-neighbor search against a Pinecone vector index containing chunked Vietnamese legal statutes and FAQ entries. The top-k most semantically similar passages — filtered by `source_type` metadata (`law` or `faq`) — are returned to the LLM as contextual evidence, and the system prompt explicitly instructs the model to cite specific articles, clauses, and legislative sources rather than fabricating legal provisions. This retrieval step ensures that the chatbot's answers remain faithful to the statutory text and are verifiable by the end user.

## 3. Case-Based Reasoning (CBR)

Case-Based Reasoning is an AI methodology in which new problems are solved by retrieving and adapting solutions from analogous past cases. In this project, CBR is implemented as a parallel retrieval pathway that searches a dedicated partition of the Pinecone index containing Vietnamese court precedent documents, filtered by `source_type: "precedent"`. When a user poses a legal question, the system executes both the RAG search over statutory texts and the CBR search over court precedents concurrently via `Promise.all`, returning up to three precedent cases alongside the statutory provisions. The system prompt then guides the LLM to perform structured analogical analysis — identifying factual similarities between the user's situation and the retrieved case, summarizing the court's legal reasoning, and drawing practical implications — thereby enriching the chatbot's response with jurisprudential depth beyond what statute retrieval alone can provide.

## 4. Function Calling

Function Calling is a mechanism provided by OpenAI's chat completion API that enables the LLM to autonomously invoke externally defined tools during inference, effectively allowing the model to decide when and how to retrieve information. In this system, a `search_legal` tool is registered with the API using a JSON schema that specifies its parameters (`query`, `source_type`, `top_k`), and the model is configured with `toolChoice: "required"` to ensure that at least one retrieval call is made before generating a response. The orchestration layer implements an iterative tool-calling loop (up to three iterations) in which the model's tool invocation requests are intercepted, the corresponding executor performs the CBR+RAG retrieval against Pinecone, and the structured results are fed back into the conversation as tool-role messages for the model to synthesize into a final answer. This architecture decouples retrieval logic from the LLM, enabling modular extension with additional tools without modifying the core inference pipeline.
